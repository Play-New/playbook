# Decisions Template

Generated by `/super:strategy`. Lives at `.superskills/decisions.md` in the user's project. Append-only log. Strategy reviews, architecture decisions, and build decisions accumulate here over time.

---

```markdown
# Architecture Decisions

## Active

Decisions that currently influence the codebase. Every active decision should be traceable to code that exists today.

[entries appended by /super:strategy, /super:build, /super:design, /super:review]

## Archived

Decisions that were superseded, reversed, or no longer apply. Moved here during `/super:strategy` refresh when the context that justified them has changed. Kept for history — understanding why something was decided matters even after the decision changes.

[entries moved here during strategy refresh]
```

## Entry Format

Each entry follows this format:

```markdown
### [Date] — [Title]
**Type:** decision | alignment-check | drift-warning | opportunity | build | research
**Status:** active | archived
**Summary:** one sentence
**EIID Layer:** Enrichment | Inference | Interpretation | Delivery | none
**Action:** what was done (or "None needed, aligned")
**Archived:** [date, reason] (only if status is archived)
```

The `build` type is used by `/super:build` when a piece is constructed. It records what was built, the implementation approach, and any deviations from the strategy.

During `/super:strategy` refresh: scan active decisions. For each, check if the context that justified it still holds. If not, move to Archived with the date and reason. This keeps the active section focused on what matters now.
